Essentia User and Developer Manual

What is Essentia?¶

Essentia is an audio analysis tool. It provides a reusable collection of algorithms and descriptors mainly used to extract features from audio files. It is written in C++, but also provides Python bindings. This means that you can control Essentia from within Python and write scripts that use Essentia’s features (just as you would in Matlab). There is also a streaming mode in which you connect algorithms and run them automatically instead of specifying explicitly the order of execution (such as PureData, Max/MSP, ...). The advantage of the streaming mode is that it requires less boilerplate code and uses less memory.

Essentia is not a framework, but rather a collection of algorithms (plus some infrastructure for multithreading and low memory usage) wrapped in a library. It doesn’t provide common high-level logic for descriptor computation (so you aren’t locked into a certain way of doing things). It rather focuses on the robustness, performance and optimality of the provided algorithms, as well as ease of use. The flow of the analysis is decided and implemented by the user, while Essentia is taking care of the implementation details of the algorithms being used. An example extractor is provided, but it should be considered as an example only, not “the” only correct way of doing things.

Essentia currently features the following algorithms and tools (among others):

  • Audio file input/output: ability to read and write nearly all audio file formats (wav, mp3, ogg, flac, etc.)
  • Standard signal processing blocks: FFT, DCT, frame cutter, windowing, envelope, smoothing
  • Statistical descriptors: mean, variance, geometric mean, moments and central moments, spread, kurtosis, skewness, flatness
  • Temporal descriptors: duration, LARM, Leq, log attack time, loudness, LPC, zero-crossing rate
  • Spectral descriptors: Bark bands, MFCC, spectral peaks, rolloff, HFC, HPCP, dissonance, tonality
  • Filters (FIR & IIR): low/high/band pass, band reject, DC removal, equal loudness
  • High-level descriptors: danceability, dynamic complexity, intensity, onset detector, key, tuning frequency, BPM, beat detection, SVM classifier
